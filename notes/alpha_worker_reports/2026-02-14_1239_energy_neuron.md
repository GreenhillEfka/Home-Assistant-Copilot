# Energy Neuron Implementation Plan (Option F)

## Ziel
Energy Monitoring und Anomalie-Erkennung für HA:
- Energy-Anomalien (ungewöhnliche Verbrauchsmuster)
- Load Shifting Opportunities (günstige Zeiten für Geräte)
- Explainability (Warum werden Energy-Vorschläge gemacht?)
- Integration mit existierenden Neurons (SystemHealth, Mood)

## Features
- GET /api/v1/energy → Full snapshot
- GET /api/v1/energy/anomalies → Unusual consumption
- GET /api/v1/energy/shifting → Load shifting opportunities
- GET /api/v1/energy/explain/{suggestion_id} → Why this suggestion
- GET /api/v1/energy/baselines → Consumption baselines
- GET /api/v1/energy/suppress → Suggestion suppression check

## Use Cases
- "Deine Waschmaschine verbraucht 40% mehr als üblich"
